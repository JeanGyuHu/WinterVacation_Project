# Your Own Portfolio (YOP)

> ### Based on Github

- ### 개요

  > ###  Github 주소를 통해서 나만의 포트폴리오(WEB)를 만들어주는 서비스

  - **기존 서비스**가 존재할 수 있으니, 조사한 결과 아직은 **없었음** (하지만, 더 상세하게 찾아볼 예정)

  ----

    - 해당 프로젝트에서 중요한 Point 는 Github의 데이터
    - 깃허브를 관리하지 않는 개발자는 대략 90%
    - 해당 프로젝트의 정체성이 흔들렸음
    - Github의 데이터를 받아올 수 없다면, Github의 데이터가 쓸모가 없다면
    - 결국은 사용자 포트폴리오를 만들어주는 웹사이트가 될 것임
    - 그래서 시장조사를 한 결과
    - 가장 큰 웹사이트 템플릿 제작사이트로 Wix 가 존재했음
      - [UX/UI 디자이너 템플릿](UX/UI 디자이너 템플릿)
      - 여기에 대한 부분은 광고라는 단점이 존재함
    - 일단은 나머지 10% 사용자 (즉, Github를 사용하는 개발자)
      - 더 나아가서 StackOverFlow 의 개발자
    - 들을 위한 프로젝트를 시작해도 전혀 작은 시장이 아님
    - 그러면 다시 생각해야 할 것이 개발자의 이력서는 무슨 정보가 포함되어야 하는가
    - 정말 개발자를 위한 포트폴리오를 만들 것이라면 무슨 정보가 필요할지
    - 이것을 **조금 더** 간편하게 만들어 주는 프로젝트여도 가능성이 있다고 판단
    - **우아한 형제들에서의 개발자 이력서라면 괜찮지 않을까 생각**
      - [이직초보 어느 개발자의 이력서 만들기](http://woowabros.github.io/experience/2017/07/17/resume.html)
      - 인상적인 첫 페이지 (My Dev Life)
        - 첫페이지는 소중하니까
      - 인적사항
        - 나를 표현하는 한 단락, 그리고 한 줄
      - 필수적인 것만
      - 독자가 관심 있어 할 내용 순으로
        - 보유 기술
        - 프로젝트 1 (최근)
        - 프로젝트 2
        - 강의 경력
        - 대학원 연구과제
        - 대외 공모전 경험
        - 대학 때의 토이 프로젝트
      - 문제 해결 및 개선 사례
    - 물론, 해당 프로젝트를 만들고 나서도 어느 사용자는 채우지 못할 수도 있음
      - 이 부분에 대한 유연성을 고려할 것인가
      - 아니면 그 사용자에게 목표가 될 것인가
      - 어떤 사람들은 아예 정보를 입력하고 싶지 않을 수 있음
  - [신입/취준생 분들을 위한 포트폴리오 제안](https://okky.kr/article/360170)
  - [개발자 포트폴리오](https://okky.kr/article/301900)

- ### Final Layout

  ![](https://i.imgur.com/6mubYcc.png)

  - [GitHub Developer API](https://developer.github.com/v3/users/)
    - Response 에서 사용자의 정보를 받아올 수 있긴 함
  - 원래는 intro 페이지를 먼저 완성하려고 했었으나, 내가 최종적으로 사용자에게 보여줄 페이지를 완성
  - 즉, 최종페이지를 먼저 완성하고 그에 살을 붙여나가는 식 (그 살은 intro 페이지가 될 것) 으로 진행
  - 최종페이지에 대해 기술적으로 먼저 구현하는 것이 목표
  - 사용자의 능력치를 뿌려주는 부분에서의 Analysis 는 아직 고려단계
    - 100%를 정의할 수 있는 부분이 아님
    - 사용자의 데이터를 가져온다고 해도 기준이 될 수 있는 부분이 존재하지 않음
  - 사용자의 능력치를 보여주는 것이 아닌 **Github의 가시성을 개선하는 부분으로 수정**
    - 예를들어, 프로젝트 마다의 사용언어 %를 그래프로 출력해준다던가
    - Java로 한 프로젝트는 몇개 (Repository 에 기반하여)
    - 이렇게 데이터를 활용할 수 있는 부분에 한해서, 가시성을 개선하는 것은 어떨까?
  - 프로젝트를 처음 시작하는 단계에서 예측의 어려움이 존재
  - 월요일마다 보고가 아닌, 1일단위나 2일단위 정도로 보고체계를 수정
    - 작은 규모로 진행하는 단계로 수정
